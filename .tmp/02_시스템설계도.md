# [버스기사다~얍] MVP 시스템 설계도

## 1. 기술 스택 (Tech Stack)
- **Runtime**: Browser (Chrome/WebView 최적화)
- **Language**: Vanilla JavaScript (ES6+), HTML5, CSS3
- **Styling**: Vanilla CSS (Tailwind를 지양하고 직접 최적화된 스타일 시트 작성)
- **Storage**: Browser LocalStorage (8GB RAM 환경을 고려한 인메모리 최소화 및 영속성 확보)
- **Deployment**: Single Page Application (SPA) 구조로 페이지 전환 부하 제거

## 2. 폴더 구조 (Folder Structure)
- `/` (Root)
    - `index.html`: 메인 프레임 및 화면 컨테이너
    - `src/`
        - `css/`
            - `main.css`: The Command Cockpit 디자인 시스템 구현
        - `js/`
            - `main.js`: 라우팅 및 초기화 로직
            - `calendar.js`: 근무 캘린더 모듈
            - `dispatch.js`: 배차표 입력 목업 모듈
            - `checklist.js`: 안전 점검 리스트 모듈
            - `storage.js`: LocalStorage 데이터 관리 유틸리티
- `.tmp/`: 프로젝트 기획/설계 산출물 보관

## 3. 핵심 아키텍처 (Core Architecture)
- **State-Driven UI**: 캘린더와 점검 리스트의 상태가 변경될 때마다 LocalStorage와 동기화하고 UI를 부분 업데이트.
- **Progressive Activation**: 1단계 범위(Active) 외의 기능은 '준비 중' 모달 또는 텍스트로 인터랙션을 차단하여 불필요한 스크립트 실행 방지.

## 4. 데이터 흐름 (Data Flow)
1. 사용자가 캘린더에서 AM/PM 클릭
2. `storage.js`를 통해 LocalStorage의 `bus_work_logs` 키에 `{date: "2024-05-19", status: "AM"}` 저장
3. 저장 완료 후 캘린더 색상 업데이트 (Red/Green)
